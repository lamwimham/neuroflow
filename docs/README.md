## 🔄 文档维护

### 更新频率

- **项目信息**: 功能更新时
- **架构设计**: 重大重构时
- **计划文档**: 每双周迭代
- **报告文档**: 每个阶段完成时
- **指南文档**: API 变更时

### 命名规范

- 文件名使用大写英文和下划线
- 例如：`FEATURE_NAME.md`, `PHASE_REPORT.md`
- 日期格式：YYYY-MM-DD

### 版本控制

所有文档都通过 Git 进行版本控制，可以通过以下命令查看历史：

```bash
# 查看文档修改历史
git log docs/

# 查看特定文档历史
git log docs/plans/ITERATION_PLAN.md
```
