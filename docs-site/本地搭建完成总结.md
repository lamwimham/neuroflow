## 🚀 常用命令

### 启动预览服务器（当前已运行）
```bash
cd /Users/lianwenhua/indie/NeuroFlow/docs-site
mkdocs serve
```

### 构建静态网站
```bash
cd /Users/lianwenhua/indie/NeuroFlow/docs-site
mkdocs build
# 生成的网站在 site/ 目录
```

### 清理并重新构建
```bash
mkdocs build --clean
```

### 部署到 GitHub Pages
```bash
# 首次部署
mike deploy --push main

# 发布新版本
mike deploy --push v0.3.0

# 设置默认版本
mike set-default --push main
```

## ⚠️ 注意事项

### 1. 预览服务器

预览服务器当前在后台运行（PID: 82063）。

**停止服务器**:
```bash
kill 82063
```

**或在前台重启**:
```bash
mkdocs serve
```

### 2. 实时重载

MkDocs 支持实时重载，当你编辑 `.md` 文件时，浏览器会自动刷新。

### 3. 缺失文件警告

构建时有一些警告，关于 nav 配置中引用但不存在的文件：
- `getting-started/next-steps.md`
- `best-practices/performance.md`
- `best-practices/security.md`
- `troubleshooting/installation.md`
- 等等...

这些是可选的进阶内容，不影响网站正常运行。
